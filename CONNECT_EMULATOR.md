# Инструкция по подключению эмулятора

## Android эмулятор (Windows)

### 1. Установка Android Studio
1. Скачайте: https://developer.android.com/studio
2. Установите с компонентами:
   - Android SDK
   - Android SDK Platform
   - Android Virtual Device (AVD)

### 2. Создание виртуального устройства
1. Откройте Android Studio
2. **Tools** → **Device Manager**
3. Нажмите **Create Device**
4. Выберите устройство (например, Pixel 5)
5. Выберите системный образ (API 33 или 34)
6. Завершите создание

### 3. Запуск эмулятора
1. В Device Manager нажмите ▶️ на устройстве
2. Дождитесь полной загрузки (может занять 1-2 минуты)

### 4. Подключение к Expo
Когда Expo сервер запущен (`npx expo start`):
- Нажмите **`a`** в терминале Expo
- Или отсканируйте QR-код в Expo Go

## Физическое устройство (проще)

### Android:
1. Установите **Expo Go** из Google Play
2. Убедитесь, что телефон и компьютер в одной Wi-Fi сети
3. В терминале Expo нажмите **`a`** или отсканируйте QR-код

### iOS (только Mac):
1. Установите **Expo Go** из App Store
2. Убедитесь, что устройства в одной Wi-Fi сети
3. В терминале Expo нажмите **`i`** или отсканируйте QR-код

## Веб-версия (самый простой)
- Нажмите **`w`** в терминале Expo
- Или откройте `http://localhost:8081` в браузере

## Проверка подключения

### Проверить, видит ли Expo эмулятор:
```bash
npx expo start
# Затем нажмите 'a' для Android
```

### Если эмулятор не виден:
1. Убедитесь, что эмулятор полностью загружен
2. Проверьте, что ADB видит устройство:
   ```bash
   adb devices
   ```
3. Перезапустите Expo сервер:
   ```bash
   npx expo start --clear
   ```

## Решение проблем

### Проблема: "No devices found"
**Решение:**
1. Убедитесь, что эмулятор запущен
2. Проверьте ADB: `adb devices` (должно показать устройство)
3. Перезапустите ADB: `adb kill-server && adb start-server`

### Проблема: "Unable to connect to Metro"
**Решение:**
1. Проверьте, что Metro сервер запущен
2. Убедитесь, что порт 8081 не занят
3. Попробуйте: `npx expo start --clear`

### Проблема: Эмулятор медленный
**Решение:**
1. В настройках AVD увеличьте RAM (минимум 2GB)
2. Включите Hardware Acceleration (HAXM или Hyper-V)
3. Используйте x86/x86_64 образ вместо ARM

## Альтернатива: Expo Go на телефоне
Самый простой способ для тестирования:
1. Установите Expo Go на телефон
2. Запустите `npx expo start`
3. Отсканируйте QR-код в приложении Expo Go
